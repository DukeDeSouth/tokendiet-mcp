## Methodology

### What we measure

- **Baseline tokens:** BPE count (`o200k_base` via `gpt-tokenizer`) of raw content that would enter agent context without compression.
- **Compressed tokens:** BPE count of tool responses actually returned (`read`/`run` JSON `content` field, or `0` for `unchanged` cache hits).
- **Session savings:** `1 - compressed/baseline` on compressible operations only.
- **Acceptance rate:** `tokendiet_reads / total_file_reads` in agent session (harness = 100%; live sessions from transcript).

### Accounting rules (v2, post P0/P1)

1. **`unchanged` only counts when the agent already saw the file in the same MCP process** (`servedThisSession`). A new chat / fresh MCP process always gets full or compressed content on first read — no fake 100% savings from SQLite cross-session cache.
2. **`saved` may be negative** — recorded honestly in stats (e.g. diff costing more than the file). P1-4 falls back to full/compressed when diff is not worthwhile (`diffWorthwhileRatio = 1`).
3. **`stats.session`** is per MCP process (`session_id`), not all-time. Use `stats.all_time` for cumulative history.
4. **Harness** replays scenarios via `handleRead`/`handleRun` with in-memory stats — same code path as live MCP, isolated `sessionId` per run.

### Runs

- **Run A (baseline):** Built-in Read/Bash returns full raw text every call.
- **Run B (TokenDiet):** `read`/`run` with compression; collect per-op `baseline` vs `compressed` and `stats.session`.

All v2 harness commands: `npm run dogfood:all` (from `tokendiet-mcp/`).

## Gate decision (v2)

### Real-session aggregate (R1 + R2 intra + R3, token-weighted)

| Scenario | Baseline | Saved | Saved % |
|----------|----------|-------|---------|
| R1-planning-v2 | 11 639 | 3 545 | 30% |
| R2-intra-session-v2 | 95 515 | 47 351 | 50% |
| R3-test-build-v2 | 9 998 | 3 553 | 36% |
| **Weighted total** | **117 152** | **54 449** | **46%** |

Real corpus share of weighted baseline: 100% (synthetic excluded from gate).

| Metric | Target | v2 result | Pass? |
|--------|--------|-----------|-------|
| Session savings (real, weighted) | ≥15% | **46%** | ✅ |
| Session savings (cold-first only, R2) | ≥15% | 5% | ❌ |
| Acceptance (harness) | ≥50% | 100% | ✅ |

**Decision: Sprint 1 gate PASS** on honest real-session aggregate (46% ≥ 15%).

**Caveats:**

1. **Cold-first workflows** (no re-reads in session) save ~5% — mostly `run` compression. Gate fails on R2-cold alone.
2. **Re-reads within session** are the main read-side saver (unchanged = 0 tokens out, counted honestly).
3. **`run` compression** reliably delivers 69–98% on test output and identical logs regardless of cache.
4. v1 R2 “88% warm” is **invalid** for gate — cross-session cache artifact; do not use for comparisons.

## v1 → v2 changelog

| Item | v1 (pre-fix) | v2 (post-fix) |
|------|--------------|---------------|
| Cross-session `unchanged` | counted as 100% saved | first read in new process = full/compressed |
| `stats.session` | all-time mix | per MCP `session_id` |
| Expensive diff | returned, `saved` clamped to 0 | fallback to compressed; honest negative if forced |
| `run` false test_output | silent line loss | strict markers + omitted_lines |
| R2 warm savings | 88% | 50% (intra-session) |
