## Качество (из живых сессий L1/L2)

- Две исследовательские задачи по незнакомым файлам решены только по сжатым ответам; контрольный `expand` подтвердил дословное совпадение outline/symbol с исходником.
- Ложных выводов сжатие не спровоцировало; эскалация понадобилась 1 раз (контрольная сверка).
- Открытая мелочь: outline не показывает top-level константы (кандидат в бэклог).

## Sanity-примечания

- Search-подсценарии SF: крупные запросы 14–61% saved, мелкие — passthrough 0% (честно); fetch: MDN −91%, JSON API и plain README — passthrough 0% (ожидаемо, JSON-схемы для API — бэклог Sprint 4).
- `stats.session` на свежем процессе стартует с нулей; история в `month`/`all_time`.
- Замечено в L2: Cursor переиспользует процесс сервера после `npm run build` — для подхвата новой сборки нужен reload MCP (запомнить для README/troubleshooting в Sprint 3 Workstream D).
