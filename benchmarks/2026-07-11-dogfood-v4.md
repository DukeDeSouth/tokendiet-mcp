## Fix verification

| Fix | Check | Result |
|---|---|---|
| glob basename (`*.ts`) | live search + `glob:"*.ts"` | 56 hits / 19 files (was **0**); rg and fallback agree |
| `.gitignore` filter | search without glob | junk from `.dogfood-td/refs` gone; 224 → 71 real hits |
| search passthrough | small result sets (≤300 BPE baseline) | `raw-passthrough`, saved 0 |
| urls-preserved (code) | outline of file with URL in function body | verified=true, no rollback |
| run omitted marker | `npm test` via run | `[omitted N non-failure lines — expand(ref)]`, 79% saved |

## Quality (live sessions L1/L2)

- Two exploration tasks on unfamiliar files solved from compressed views only; `expand` confirmed outline/symbol matched source.
- No false conclusions from compression; one escalation for control verification.
- Known gap: outline omits some top-level constants (backlog).

## Notes

- SF search: large queries 14–61% saved, small — passthrough 0%; fetch: MDN −91%, small JSON/README — passthrough 0%.
- `stats.session` resets on fresh MCP process; use `month` / `all_time` for history.
- After `npm run build`, reload MCP in Cursor so the server picks up the new `dist/` (process reuse).
